    Service Entrypoint

    pydsf.DSFService

    Main service entrypoint for managing, querying, and verifying datasets.

    Provides core concurrency fencing (mark_status), cryptographic hashing (update_merkle), and topological DAG verification across storage backends.

    Example

    python service = DSFService() # Erect fence -> mutate disk -> seal changes -> tear down fence service.mark_status("data@v1", BusyStatus.Modifying) # ... modify disk files ... service.update_merkle("data@v1") service.mark_status("data@v1", BusyStatus.Free)

    verify_deep(id, show_diff=False, target_backend=None)

    Performs deep topological verification of target and its dependency DAG.

    Respects concurrency fences: if any upstream dependency is marked with a non-read busy status, verification short-circuits to report the lock without hashing.

    Parameters:

    id str

    		 False
    target_backend Optional[BackendAddr]

    Optional specific backend to run verification against.

    		 None

    Returns:

    Type Description

    Type Description

    verify_self(id, show_diff=False, target_backend=None)

    Performs verification strictly on target dataset, ignoring upstream states.

    pydsf.DatasetStatus

    Represents the overall health or verification status of a dataset.

    Attributes:

    Name Type Description
    Healthy DatasetStatus

    Dataset is intact, verified, and free of corruption.
    Broken DatasetStatus

    Dataset files have failed verification or are missing.
    BrokenDeps DatasetStatus

    Target dataset is fine, but one or more upstream dependencies are broken.
    Unverified DatasetStatus

    Dataset has been registered or modified but not yet verified.
